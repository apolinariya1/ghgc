 1. Введение

 1.1. Общая информация о проекте

Данная документация разработана для пояснения как работает проект, который представляет собой настольное приложение под названием "Управлятор задач" (Task Manager), предназначенное для эффективного ведения и управления личными и рабочими задачами.

 1.2. Цели и задачи разработки

Основной целью разработки приложения является создание удобного и надежного инструмента для организации повседневных дел, предотвращения забывчивости и повышения личной продуктивности. Для достижения этой цели были поставлены следующие задачи:
•   Реализация механизма авторизации и регистрации для поддержки многопользовательского режима.
•   Разработка интуитивно понятного графического интерфейса пользователя.
•   Обеспечение функциональности по добавлению, редактированию, удалению и отметке задач как выполненных.
•   Внедрение системы статусов и приоритетов для гибкого управления задачами.
•   Организация надежного хранения данных с использованием системы управления базами данных.

1.3. Принципы разработки

В процессе разработки приложения "Управлятор задач" были применены следующие принципы:
•  Удобство использования: Интерфейс приложения спроектирован с учетом максимальной простоты и интуитивности для пользователей любого уровня подготовки.
•  Функциональность: Реализованный набор функций направлен на удовлетворение базовых потребностей пользователя в управлении задачами.
•  Надёжность: Особое внимание уделено сохранности пользовательских данных и стабильности работы приложения.


 2. Актуальность и решаемые проблемы

 2.1. Обоснование необходимости приложения

В современном ритме жизни человек ежедневно сталкивается с большим объемом информации и множеством задач, требующих выполнения. Неэффективные методы их учёта (бумажные записи, стандартные заметки) часто приводят к забывчивости, потере информации и снижению общей продуктивности. Данное приложение призвано решить эту проблему, предоставляя централизованный, структурированный и надежный способ управления задачами.

 2.2. Ключевая функциональность приложения

Приложение "Управлятор задач" обеспечивает следующую функциональность:
•  Многопользовательский режим с разделением данных между аккаунтами.
•  Визуальная индикация срочности и важности задач.
•  Отслеживание выполнения задач с помощью системы статусов.
•  Возможность сортировки задач по различным критериям, включая приоритет.
•  Долгосрочное и надежное хранение данных, устойчивое к сбоям системы.

 2.3. Целевая аудитория и сценарии использования

Приложение имеет универсальный характер и предназначено для широкого круга пользователей, сталкивающихся с необходимостью планирования и учёта своих дел

 3. Архитектура системы

Приложение организовано по модульному принципу, обеспечивающему разделение функциональности.

 3.1. Модуль авторизации и регистрации

Модуль авторизации и регистрации обеспечивает персонализированный доступ к данным пользователей и их конфиденциальность.

 3.1.1. Механизм авторизации

Система авторизации позволяет пользователям получать доступ к своим задачам после успешной идентификации.

Процесс авторизации:
1. Пользователь вводит имя пользователя (логин) и пароль в соответствующие поля.
2. Система осуществляет проверку введенных учетных данных на соответствие записям в базе данных.
3. При успешном совпадении логина и хеша пароля, пользователь получает доступ к главному окну приложения.
4. В случае некорректных данных, выдается сообщение об ошибке.

Для обеспечения безопасности, поле ввода пароля скрывает вводимые символы.

 3.1.2. Процесс регистрации нового пользователя

Для новых пользователей предусмотрена функция регистрации, позволяющая создать новую учетную запись.

Процесс регистрации:
1. Пользователь заполняет поля "Логин", "Email", "Пароль" и "Повтор пароля".
2. Система выполняет валидацию введенных данных:
  •  Уникальность логина: Проверяется отсутствие логина в базе данных.
  •  Формат Email: Проверяется корректность формата электронной почты (наличие символа '@').
  •  Сложность пароля: Пароль должен содержать не менее 3 символов.
  •  Подтверждение пароля: Введенные пароли в полях "Пароль" и "Повтор пароля" должны совпадать.
3. При успешной валидации данные сохраняются в базе данных, и пользователь может осуществить вход.

Поле Email предусмотрено для будущих функциональных расширений, таких как восстановление пароля или система уведомлений.

 3.2. Основной интерфейс и управление задачами

После успешной авторизации пользователь переходит в главное окно приложения, которое предоставляет централизованный доступ к управлению задачами.

Основные элементы главного окна:
•  Панель приветствия: Отображает имя текущего пользователя, подтверждая успешный вход. Содержит кнопку "Выйти" для завершения сеанса.
•  Панель статистики: Представляет сводную информацию о задачах: общее количество, количество новых, задач в работе и завершенных.
•  Панель управления задачами: Набор кнопок для выполнения базовых операций:
  •  Кнопка "Все" для фильтрации задач.
  •  Кнопка "Обновить" для обновления списка задач.
  •  Кнопка "Добавить" для создания новой задачи.
  •  Кнопка "Редактировать" для изменения выбранной задачи.
  •  Кнопка "Удалить" для удаления выбранной задачи.
  •  Кнопка "Выполнено" для отметки задачи как завершенной.
•  Список задач: Табличное представление всех задач пользователя с ключевой информацией (ID, Название, Статус, Важность, Срок).
•  Визуальная индикация: Цветовые метки (например, красный, оранжевый, зеленый) рядом с задачами, обозначающие их приоритет или срочность.

 3.3. Структура данных задачи

Каждая задача в системе представлена набором атрибутов, обеспечивающих её полную детализацию и эффективное управление
Данная структура позволяет полно описывать каждую задачу, обеспечивая ее однозначную интерпретацию и отслеживание на всех этапах жизненного цикла.

 3.4. Управление статусами и приоритетами задач

Для гибкого отслеживания прогресса и определения срочности выполнения задач реализованы механизмы статусов и приоритетов.

 Статусы задач

В приложении определены три основных статуса, которые охватывают весь жизненный цикл задачи
Система статусов позволяет пользователю быстро оценить текущее состояние своих дел и фокусироваться на задачах, требующих внимания.

 Приоритеты задач

Для определения срочности и важности задач реализована трехуровневая система приоритетов, цветовая индикация приоритетов в списке задач обеспечивает быстрое визуальное восприятие их важности, позволяя пользователю оперативно расставлять акценты в планировании.

4. Архитектура хранения данных

 4.1. Обоснование выбора СУБД

Для обеспечения надежного, структурированного и эффективного хранения данных было принято решение использовать систему управления базами данных (СУБД) Microsoft SQL Server. Выбор в пользу СУБД обусловлен следующими преимуществами перед файловым хранением:
•  Целостность данных: СУБД обеспечивает механизмы для поддержания непротиворечивости данных.
•  Многопользовательский доступ: Поддержка одновременной работы нескольких пользователей без конфликтов.
•  Производительность: Оптимизированные механизмы поиска и выборки данных.
•  Масштабируемость: Возможность расширения объёма данных и количества пользователей.
•  Безопасность: Инструменты для разграничения прав доступа и защиты информации.
•  Устойчивость к сбоям: Механизмы транзакций и восстановления данных.

 4.2. Структура базы данных

База данных приложения состоит из двух основных таблиц, связанных по принципу "один ко многим":

Таблица USERS:
•  UserID (Целочисленный, Первичный ключ) - Уникальный идентификатор пользователя.
•  Username (Текстовый) - Имя пользователя (логин).
•  Email (Текстовый) - Адрес электронной почты.
•  PasswordHash (Текстовый) - Хеш пароля.
•  CreatedAt (Дата/Время) - Дата регистрации.

Таблица TASKS:
•  TaskID (Целочисленный, Первичный ключ) - Уникальный идентификатор задачи.
•  UserID (Целочисленный, Внешний ключ) - Владелец задачи, ссылка на USERS.UserID.
•  Title (Текстовый) - Название задачи.
•  Description (Текстовый, может быть многострочным) - Детальное описание.
•  Status (Текстовый) - Статус задачи (например, "Новая", "В работе", "Завершена").
•  Priority (Целочисленный) - Приоритет задачи (1-3).
•  DueDate (Дата) - Срок выполнения.
•  CreatedAt (Дата/Время) - Дата создания задачи.
•  CompletedAt (Дата/Время, может быть NULL) - Дата завершения задачи.

Связь между таблицами:
Таблица USERS связана с таблицей TASKS по принципу "один ко многим". Одному пользователю (одна запись в таблице USERS) может принадлежать множество задач (несколько записей в таблице TASKS). Эта связь осуществляется через поле UserID в обеих таблицах (USERS.UserID - первичный ключ, TASKS.UserID - внешний ключ). Эта структура гарантирует, что задачи разных пользователей хранятся изолированно и не могут быть случайно перемешаны.

 7. Приложение: Визуализация элементов интерфейса и структуры данных

 7.1. Модуль авторизации: Вход в систему

Окно авторизации включает следующие элементы:
•  Заголовок окна: "ВХОД В СИСТЕМУ".
•  Поле ввода для логина пользователя.
•  Поле ввода для пароля пользователя (скрывает символы).
•  Кнопка "Войти" для осуществления входа.
•  Кнопка "Регистрация" для перехода к созданию новой учетной записи.
•  Кнопка "Настройки" для доступа к параметрам приложения.

 7.2. Модуль регистрации: Создание нового пользователя

Окно регистрации включает следующие элементы:
•  Заголовок окна: "РЕГИСТРАЦИЯ".
•  Поле ввода для выбора логина.
•  Поле ввода для адреса электронной почты (Email).
•  Поле ввода для установки пароля (скрывает символы).
•  Поле ввода для подтверждения пароля (повторный ввод).
•  Кнопка "Зарегистрироваться" для создания учетной записи.

 7.3. Основной интерфейс: Главное окно приложения

Главное окно приложения имеет следующую структуру:
•  Верхняя панель:
  •  Приветствие пользователя (например, "Добро пожаловать, Иван!").
  •  Кнопка "Выйти".
•  Панель статистики:
  •  Информация о задачах: "Всего: [количество]", "Новых: [количество]", "В работе: [количество]", "Готово: [количество]".
•  Панель управления:
  •  Кнопки действий: "Все" (фильтр), "Обновить", "Добавить", "Редактировать", "Удалить", "Выполнено".
•  Таблица задач:
  •  Столбцы: "ID", "Название", "Статус", "Важность", "Срок".
  •  Примеры строк:
    *  "ID: 1 | Название: Купить молоко | Статус: Новая | Важность: Низкая | Срок: 01.12.2024" (индикация: Зеленый)
    *  "ID: 2 | Название: Сделать отчёт | Статус: В работе | Важность: Средняя | Срок: 28.11.2024" (индикация: Оранжевый)
    *  "ID: 3 | Название: Запись к врачу | Статус: Новая | Важность: Высокая | Срок: 25.11.2024" (индикация: Красный)
    *  "ID: 4 | Название: Позвонить маме | Статус: Завершена | Важность: Средняя | Срок: 20.11.2024" (индикация: Нет)

 7.4. Детализация задачи

Окно просмотра/редактирования отдельной задачи (например, "ЗАДАЧА №42") содержит следующие поля с данными:
•  "Название: Позвонить маме"
•  "Описание: Поздравить с днём рождения"
•  "Статус: Новая"
•  "Приоритет: Высокий (3)"
•  "Срок: 25.11.2024"
•  "Создано: 20.11.2024"
•  "Владелец: Иван"
